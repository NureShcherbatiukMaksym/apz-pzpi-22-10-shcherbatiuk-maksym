Харків 2025
     3.1 Опис застосунку
     Для реалізації front-end частини системи використано React.js з Tailwind CSS для адаптивного інтерфейсу. Компоненти побудовано за принципами компонентного підходу з використанням React Router для маршрутизації та Context API для глобального стану, включно з авторизацією і мовними налаштуваннями. Аутентифікація реалізована через Firebase (електронна пошта або Google-акаунт), з підтримкою ролей для розділення доступу. Інтерфейс локалізовано за допомогою i18next (українська/англійська), з підтримкою регіонального формату дат і сортування. Аналітика візуалізується через Chart.js. Адміністративна частина підтримує керування користувачами, пристроями та налаштуваннями з можливістю експорту/імпорту. Комунікація з бекендом здійснюється через REST API у форматі JSON.
     
     3.2 Взаємодія компонентів
     Застосунок побудований як набір React-компонентів, які взаємодіють через пропси та Context API. Для навігації використовується React Router. Дані отримуються з бекенду через axios (REST API) та відображаються в інтерфейсі (таблиці, графіки, форми). Авторизація обробляється через Firebase, з передачею токена для захищених запитів. Інтерфейс динамічно адаптується під роль користувача. Уся система узгоджено працює з бекендом, забезпечуючи інтерактивність і реактивність.
